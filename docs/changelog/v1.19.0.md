# v1.19.0

## FormKiQ Core (v1.19.0)

### Feature

- [#450](https://github.com/formkiq/formkiq-core/issues/450) Added `filename` and `folder` search indexes to `POST /search`, enabling folder-name searches and queries that list all files in a folder
- [#480](https://github.com/formkiq/formkiq-core/issues/480) Added artifact-enabled documents, including artifact metadata, artifact categories, `hasArtifacts`, document artifact promotion, and artifact-aware upload, URL, OCR, tags, attributes, restore, purge, retry, reindex, and workflow support
- [#484](https://github.com/formkiq/formkiq-core/issues/484) Added start and end date support when searching soft-deleted documents
- [#489](https://github.com/formkiq/formkiq-core/issues/489) Added SSO `automaticSignIn` to system configuration
- [#473](https://github.com/formkiq/formkiq-core/issues/473) Added `CHECKSUM` document actions that calculate and persist SHA1, SHA256, or SHA512 checksums from document content, with SHA512 also supported when generating document upload URLs for S3 checksum validation
- [#496](https://github.com/formkiq/formkiq-core/issues/496) Added classification conditionals to mappings
- [#500](https://github.com/formkiq/formkiq-core/issues/500) Added `validationRegex` support for consistent document attribute values
- Added `MOVE` and `DELETE` document actions and `ASYNC_COMPLETE` action status
- Added `PUT` support for entity types and entities
- Added CloudFormation configuration for setting Cognito access token and refresh token validity
- Added NACL support to the VPC template

### Updates

- Updated Console to 4.1.1
- Updated FormKiQ client dependency to 1.19.0
- [#366](https://github.com/formkiq/formkiq-core/issues/366) Updated Java runtime to JDK 25
- Refactored OpenAPI generation into reusable templates and path files
- Refactored Gradle modules into layered directories
- Updated local Docker stack versions for Console, Keycloak, Typesense, MinIO, and DynamoDB
- Updated GitHub Actions to use OIDC
- Added Schemathesis testing
- Stopped using SSM dynamic references in CloudFormation templates
- Enabled Container Insights on the ECS cluster
- Added `userClaims`, roles, and site permission mapping details to `GET /sites`
- Added `retentionAndDisposition` to `/sites/{siteId}/configuration`
- Added `deletedDate` to soft-deleted documents and remove it on restore
- Added reserved governance and LLM attribute keys including `Sensitivity` and `LlmMaxToken`
- Improved DynamoDB contention handling and config table read caching

### Bug Fixes

- [#455](https://github.com/formkiq/formkiq-core/issues/455) Added `documentId` to GSI1 of the folder index
- [#466](https://github.com/formkiq/formkiq-core/issues/466) Fixed CloudFront configuration before the S3 bucket is populated
- [#485](https://github.com/formkiq/formkiq-core/issues/485) Fixed `DELETE /entities/{entityTypeId}/{entityId}` not-found errors returning the wrong ID
- [#507](https://github.com/formkiq/formkiq-core/issues/507) Fixed file extension detection for dotted paths without extensions
- [#509](https://github.com/formkiq/formkiq-core/issues/509) Fixed `POST /documents/compress` failures on documents with artifacts
- [#458](https://github.com/formkiq/formkiq-core/issues/458) Fixed child document tags and attributes not being saved on `POST`


## FormKiQ Platform (v1.10.0)

### Feature

- [#168](https://github.com/formkiq/formkiq-platform/issues/168) Added site delegation tokens for downscoped and on-behalf-of access through `POST /sites/{siteId}/delegationTokens`
- [#199](https://github.com/formkiq/formkiq-platform/issues/199) Added initial contract management preset entities
- [#217](https://github.com/formkiq/formkiq-platform/issues/217) Added `ASYNC_COMPLETE` workflow action status and expanded workflow attribute-comparison conditions
- [#219](https://github.com/formkiq/formkiq-platform/issues/219) Added document review handlers for `GET`, `POST`, and `PATCH /documents/{documentId}/reviews`
- [#183](https://github.com/formkiq/formkiq-platform/issues/183) Added entity-based disposition support using a new `DispositionPolicy` PRESET entity type, schema references through the `DispositionPolicy` attribute key, required pairing with `RetentionPolicy`, inherited retention date basis, and nightly scheduler execution of disposition actions
- [#180](https://github.com/formkiq/formkiq-platform/issues/180) Added the ShortLinks API, enabling applications to create managed short URLs with `POST /shortlinks`, return shortened document URLs when requesting `format=short`, and resolve public `GET /s/{slug}` links with a temporary redirect to the stored destination URL
- Added system inference model management endpoints: `GET /system/inferenceModels` lists available Bedrock-backed inference models with lifecycle, agreement, authorization, entitlement, region, and invocation metadata, while `POST /system/inferenceModels/agreement` lets administrators accept the required model usage agreement by `modelId`
- Added document AI prompt endpoints, including prompt result mapping and analysis category support

### Updates

- Added `ModelId` and `LlmMaxToken` support to LLM prompt preset entities
- Updated Platform module builds to Java 25

### Bug Fixes

- Fixed legacy User Activity document cleanup for artifact fields
- Fixed site-permissions-defined test coverage
- Fixed EventBridge Scheduler Lambda packaging location
